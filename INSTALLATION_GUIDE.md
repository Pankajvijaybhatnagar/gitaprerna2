## ğŸ”§ Understanding npm install

### What is node_modules?

- Folder containing all JavaScript libraries
- Required for the app to work
- **NOT included in download** (too large: 200MB)
- Generated by `npm install`
- Safe to delete and regenerate

### Why download separately?

1. **Size**: 200MB is too large to include
2. **Platform-specific**: Some packages compile differently on Windows/Mac/Linux
3. **Updates**: You get the latest compatible versions
4. **Standard practice**: All Node.js projects work this way

### What is package.json?

- List of dependencies and their versions
- Like a "shopping list" for npm
- npm reads this to know what to download

### What is package-lock.json?

- Exact versions of installed packages
- Auto-generated by npm
- Ensures everyone gets same versions
- Commit this to git (if using version control)

## ğŸ› Troubleshooting npm install

### Error: "npm: command not found"

**Problem:** Node.js not installed correctly

**Solution:**
1. Reinstall Node.js from nodejs.org
2. Restart your terminal/computer
3. Verify: `node --version`

### Error: "EACCES: permission denied"

**Problem:** Permission issues (Mac/Linux)

**Solution:**
```bash
sudo npm install
# Enter your password when prompted
```

### Error: "ENOENT: no such file or directory"

**Problem:** Wrong folder or corrupted files

**Solution:**
1. Verify you're in correct folder: `ls` or `dir`
2. Re-extract the zip file
3. Try again

### Error: "network timeout" or "fetch failed"

**Problem:** Internet connection issues

**Solution:**
1. Check internet connection
2. Try again: `npm install`
3. If using VPN, try without it
4. Use different network if possible

### Error: "npm ERR! code ERESOLVE"

**Problem:** Dependency conflicts

**Solution:**
```bash
npm install --legacy-peer-deps
```

### Error: Takes forever / Seems stuck

**Problem:** Slow internet or large download

**Solution:**
- Be patient! Can take 5-10 minutes on slow internet
- Check internet speed
- npm is still working if you see any activity
- Don't close terminal!

### Error: "gyp ERR!" or build errors

**Problem:** Missing build tools (usually Windows)

**Solution:**
```bash
npm install --ignore-scripts
```

**Or install Windows build tools:**
```bash
npm install -g windows-build-tools
```

## ğŸ¯ Common Questions

### Q: Do I need to install node_modules every time?

**A:** No! Only once. node_modules stays in your folder.

### Q: Can I delete node_modules?

**A:** Yes, but you'll need to run `npm install` again.

### Q: Why is node_modules so big?

**A:** It contains 300+ packages with all their code. Normal for modern web apps.

### Q: Do I commit node_modules to git?

**A:** No! It's in `.gitignore`. Only commit `package.json` and `package-lock.json`.

### Q: Can I copy node_modules between projects?

**A:** No! Each project needs its own. Don't copy, run `npm install`.

### Q: What if npm install fails completely?

**A:** Try:
```bash
# Clear npm cache
npm cache clean --force

# Try installing again
npm install
```

### Q: Can I use yarn instead of npm?

**A:** Yes!
```bash
yarn install
yarn dev
```
